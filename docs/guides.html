Token Management #

Access tokens are short-lived (1 hour) and automatically revoked when a new token is generated. Follow these best practices:

ℹ️
Cache your token. Don't generate a new token for every API call. Cache it and refresh only when it expires or you receive a 401 error.

Token Refresh Pattern

Implement a token manager that caches the token and refreshes it proactively before expiry:

JavaScript
class TokenManager {
  constructor(clientId, clientSecret) {
    this.clientId = clientId;
    this.clientSecret = clientSecret;
    this.token = null;
    this.expiresAt = null;
  }

  async getToken() {
    // Return cached token if still valid (with 5-minute buffer)
    if (this.token && this.expiresAt && Date.now() < this.expiresAt - 300000) {
      return this.token;
    }

    // Generate new token
    const response = await fetch(
      'https://staging.rewardgenix.co.in/clientapi/generate-token',
      {
        method: 'POST',
        headers: {
          'x-cx-client-id': this.clientId,
          'x-cx-client-secret': this.clientSecret
        }
      }
    );

    if (!response.ok) {
      throw new Error(`Token generation failed: ${response.status}`);
    }

    const { data } = await response.json();
    this.token = data.accessToken;
    this.expiresAt = new Date(data.expiresAt).getTime();
    return this.token;
  }

  invalidate() {
    this.token = null;
    this.expiresAt = null;
  }
}

Error Handling #

The API returns structured error responses with specific error codes. Always check both the HTTP status code and the error code in the response body.

Error Response Format

JSON
{
  "message": "Description of the error",
  "status": "Error",
  "code": "ER_XXX_XXX"
}

Recommended Error Handler

JavaScript
async function apiCall(url, options, tokenManager) {
  const token = await tokenManager.getToken();

  const response = await fetch(url, {
    ...options,
    headers: {
      ...options.headers,
      'Authorization': `Bearer ${token}`,
      'x-cx-api-secret': process.env.CX_API_SECRET
    }
  });

  // Handle expired token — refresh and retry once
  if (response.status === 401) {
    const error = await response.json();
    if (error.code === 'ER_401_101') {
      tokenManager.invalidate();
      const newToken = await tokenManager.getToken();
      options.headers['Authorization'] = `Bearer ${newToken}`;
      return fetch(url, options).then(r => r.json());
    }
    throw new Error(`Auth error: ${error.message} (${error.code})`);
  }

  // Handle rate limiting
  if (response.status === 429) {
    const retryAfter = response.headers.get('Retry-After') || 60;
    throw new Error(`Rate limited. Retry after ${retryAfter}s`);
  }

  if (!response.ok) {
    const error = await response.json();
    throw new Error(`API error ${response.status}: ${error.message} (${error.code})`);
  }

  return response.json();
}

Handling Partial Failures in Batch Operations

Batch create/update operations return 201/200 even when some items fail. Always inspect the results array:

JavaScript
const result = await apiCall(url, options, tokenManager);

const successes = result.data.results.filter(r => r.success);
const failures = result.data.results.filter(r => !r.success);

console.log(`Created: ${successes.length}, Failed: ${failures.length}`);

failures.forEach(f => {
  console.error(`  User ${f.userId}: ${f.error}`);
});

Rate Limiting #

The API enforces rate limits to ensure fair usage. When you exceed the limit, you'll receive a 429 Too Many Requests response.

Code Status Description
ERR_429_001 429 Rate limit exceeded. Check Retry-After header for wait time.

Rate Limit Response

JSON
// HTTP 429
// Headers: Retry-After: 60
{
  "message": "Rate limit exceeded. Please retry after 60 seconds.",
  "status": "Error",
  "code": "ERR_429_001"
}

Best Practices

  • Implement exponential backoff with jitter
  • Respect the Retry-After header value
  • Use batch endpoints to reduce the number of API calls
  • Queue requests on the client side to stay within limits

Batch Operations #

Both create and update endpoints support batch operations. Send multiple users in a single request to reduce API calls.

✅
Best Practice: Always use batch endpoints. Sending 100 users in one request is far more efficient than 100 individual requests.

Chunking Large Datasets

For very large datasets, split users into chunks to avoid request size limits:

JavaScript
const CHUNK_SIZE = 50;

async function bulkCreateUsers(allUsers, tokenManager) {
  const results = [];

  for (let i = 0; i < allUsers.length; i += CHUNK_SIZE) {
    const chunk = allUsers.slice(i, i + CHUNK_SIZE);

    const response = await apiCall(
      'https://staging.rewardgenix.co.in/clientapi/users',
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ users: chunk })
      },
      tokenManager
    );

    results.push(...response.data.results);

    // Small delay between chunks to respect rate limits
    if (i + CHUNK_SIZE < allUsers.length) {
      await new Promise(resolve => setTimeout(resolve, 1000));
    }
  }

  return results;
}

Security Best Practices #

Store Secrets Securely

Use environment variables or a secrets manager. Never hardcode credentials in source code.

IP Whitelisting

Configure IP whitelisting to restrict API access to known server IPs only.

Use HTTPS Only

All API calls must use HTTPS. Never transmit credentials over plain HTTP.

Rotate Credentials

Periodically rotate credentials via the super admin rotate-credential endpoint.

Environment Variables

Bash
# .env — NEVER commit this to version control
CX_CLIENT_ID=your-client-id
CX_CLIENT_SECRET=your-client-secret
CX_API_SECRET=your-api-secret
RG_BASE_URL=https://staging.rewardgenix.co.in
🚫
Never expose credentials in client-side code. The Client Exchange API is designed for server-to-server communication only.

Retry Strategy #

Implement exponential backoff with jitter for resilient API integration:

JavaScript
async function withRetry(fn, maxRetries = 3) {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      if (attempt === maxRetries) throw error;

      // Only retry on rate limit or server errors
      const isRetryable = error.message.includes('429') ||
                         error.message.includes('500') ||
                         error.message.includes('Rate limited');

      if (!isRetryable) throw error;

      // Exponential backoff with jitter
      const baseDelay = Math.pow(2, attempt) * 1000;
      const jitter = Math.random() * 1000;
      const delay = baseDelay + jitter;

      console.log(`Retry ${attempt + 1}/${maxRetries} after ${delay}ms`);
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }
}
